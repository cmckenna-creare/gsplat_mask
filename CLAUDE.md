## Project Overview

**gsplat** is a CUDA-accelerated Python library for differentiable 3D Gaussian Splatting (3DGS). It provides differentiable rasterization of 3D/2D Gaussians with Python bindings, supporting multiple rendering variants (3DGS, 2DGS, 3DGUT), arbitrary batching, and distributed training.

## Installation & Build

For CUDA kernel development (recommended — enables JIT incremental recompilation):
```bash
BUILD_NO_CUDA=1 pip install -e .[dev]
```

For pure Python development (full compile at install time):
```bash
pip install -e .[dev]
```

JIT-compiled CUDA extensions land in `~/.cache/torch_extensions/py*-cu*/`. Build parameters are cached to JSON; the system detects when recompilation is needed automatically.

**Build feature toggles (env vars):** `BUILD_3DGS`, `BUILD_2DGS`, `BUILD_3DGUT`, `BUILD_ADAM`, `BUILD_RELOC`, `BUILD_CAMERA_WRAPPERS`

**Other CUDA build flags:** `DEBUG`, `FAST_MATH` (default=1), `VERBOSE`, `MAX_JOBS`, `NUM_CHANNELS`, `NVCC_FLAGS`

**Clangd setup** (for C++/CUDA IDE support) — generate `.clangd` and `compile_commands.json`:
```bash
echo "# Autogenerated, see .clangd_template\!" > .clangd && \
  sed -e "/^#/d" -e "s|YOUR_CUDA_PATH|$(dirname $(dirname $(which nvcc)))|" .clangd_template >> .clangd
bear -- pip install -e gsplat/
```

## Commands

**Tests** (requires GPU; not run in CI due to GPU unavailability on GH Actions):
```bash
pytest tests/                         # all tests
pytest tests/test_basic.py            # single file
pytest tests/test_basic.py::test_name # single test function
```
Test functions must be named `test_*`. `pytest.ini` sets `pythonpath` and injects env vars like `NUM_CHANNELS`.

**Formatting** (checked in CI, run before committing):
```bash

```


## Architecture

### Rendering Pipeline

The main entry point is `gsplat/rendering.py::rasterization()`. It chains:
1. `fully_fused_projection` — projects 3D Gaussians to 2D screen space
2. `isect_tiles` — assigns Gaussians to screen tiles
3. `isect_offset_encode` — encodes tile intersections
4. `rasterize_to_pixels` — alpha-composites Gaussians front-to-back

`RenderMode` (string literal type) controls what is rendered — color, depth, hit distance, or combinations: `"RGB"`, `"d"`, `"D"`, `"Ed"`, `"ED"`, `"RGB-d"`, `"RGB-Ed"`, `"RGB+D"`, `"RGB+ED"`. Use `render_mode_has_color/depth/hit_distance()` helpers to query modes.

`RasterizeMode` is `"classic"` or `"antialiased"`.

### CUDA Layer

`gsplat/cuda/_wrapper.py` is the Python interface to CUDA kernels. `gsplat/cuda/_torch_impl.py` provides pure-PyTorch reference implementations of the same operations (used for testing/fallback). CUDA source lives in `gsplat/cuda/csrc/`.

Variants:
- `_torch_impl_2dgs.py` — 2D Gaussian splatting
- `_torch_impl_eval3d.py` — evaluation/inference path
- `_torch_cameras.py` — camera distortion models (FTheta, external polynomial, rolling shutter, LIDAR)

### Densification Strategies (`gsplat/strategy/`)

`Strategy` base class defines `step_pre_backward()` and `step_post_backward()` callbacks used during training. `DefaultStrategy` and `MCMCStrategy` are the two implementations. `ops.py` contains the underlying split/prune/clone operations.

### Key Modules

| Module | Purpose |
|---|---|
| `gsplat/rendering.py` | High-level differentiable rasterization API |
| `gsplat/cuda/_wrapper.py` | Python↔CUDA kernel bindings |
| `gsplat/cuda/build.py` | JIT build logic and feature flag parsing |
| `gsplat/strategy/` | Gaussian densification strategies |
| `gsplat/utils.py` | Projection matrices, depth-to-normal, export helpers |
| `gsplat/compression/` | PNG-based splat compression |
| `gsplat/optimizers/` | `SelectiveAdam` custom optimizer |
| `gsplat/distributed.py` | Multi-GPU all-gather/all-to-all utilities |
| `examples/simple_trainer.py` | Reference training script |

### Batching

Recent feature (May 2025): `rasterization()` supports arbitrary batching over multiple scenes and viewpoints via packed tensor representations with batch/camera/Gaussian ID indices. See `docs/batch.md`.
